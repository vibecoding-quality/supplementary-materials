# Sudy on code quality of vibe coded apps -- Supplementary Material

This repository contains the supplementary material accompanying the
article on **using Large Language Models (LLMs) for web application
development and evaluating the resulting software quality and
security**.

The repository includes the documentation, source code, analysis
reports, and screenshots generated during the empirical study. It is
intended to support transparency, reproducibility, and further analysis
of the experiment.

## Repository Contents

``` text
├── Participant related documnetation
│   ├── Participant interviews
│   ├── Consent letter
│   └── Participant profiles
├── Project analysis reports
│   ├── Semgrep
│   ├── Sonarcube
│   └── owasp ZAP
├── Project code
├── Project screenshots
└── README.md
```

## Description

### Participant related documnetation

Contains the documentation used during the experiment, including:

-   Participant consent letter template.
-   Structured interview questions.
-   Participant interview answers.

### Project analysis reports

Contains outputs from the automated analysis tools used
in the study.

-   **SonarQube** quality and static code analysis.
-   **Semgrep** security-focused static analysis.
-   **OWASP ZAP** dynamic application security testing.

### Project source code

Contains source code for all eight developed projects.

### Project Screenshots

Contains screenshots of all eight applications developed during the
experiment.

For each application, screenshots include:

-   Login and registration interface.
-   Main application view.
-   Demonstration of data persistence across application restarts.
